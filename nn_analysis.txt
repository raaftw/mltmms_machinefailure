Why Neural Networks Failed on This Task: Evidence from Our Experiments

We discovered that neural networks don't work for this problem by actually trying them. Running the same LSTM configuration twice on identical data gave completely different results: validation PR-AUC ranged from 0.02 to 0.22, and test recall swung from 33% to 56% on the same test set. This isn't due to bugs or poor tuning—it's because we only have 30 positive equipment failures in a validation set of 22,478 samples (0.13% positive rate). With so few failures, tiny changes in weight initialization cause the model to pick drastically different decision thresholds. This is a critical problem because high recall is essential for us: missing a failure can be costly. When we added deterministic seeding to make results consistent, performance got worse (test recall dropped to 21%, PR-AUC to 0.02). So we faced an impossible choice: either accept unreliable predictions that vary by 20+ percentage points in recall, or accept worse model quality. Tree-based methods like XGBoost naturally handle this extreme class imbalance and give stable results without this performance-stability trade-off, making them the right choice for rare-event prediction.
